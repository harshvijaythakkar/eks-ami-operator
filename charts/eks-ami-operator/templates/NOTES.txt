eks-ami-operator has been deployed to namespace {{ .Release.Namespace }}.

Check the operator pod:

  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "eks-ami-operator.name" . }}

Once the pod is running, create a NodeGroupUpgradePolicy to start managing a node group:

  kubectl apply -f - <<EOF
  apiVersion: eks.aws.harsh.dev/v1alpha1
  kind: NodeGroupUpgradePolicy
  metadata:
    name: my-nodegroup
    namespace: {{ .Release.Namespace }}
  spec:
    region: us-east-1
    clusterName: my-eks-cluster
    nodeGroupName: my-managed-nodegroup
    checkInterval: 24h
    autoUpgrade: false
  EOF

Check status:

  kubectl get nodegroupupgradepolicies -n {{ .Release.Namespace }}
  kubectl describe nodegroupupgradepolicy my-nodegroup -n {{ .Release.Namespace }}
